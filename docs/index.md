## Custom Project

### Dataset

The dataset used by my Kafka producer is `sales_sowers.csv`.

The dataset contains sales transaction records for an online learning
business. Each record includes fields such as order ID, date and time,
region ID, currency code, product ID, unit price, quantity, customer
ID, payment method, referral source, and customer notes.

I used a modified version of the original sales dataset and renamed it
to `sales_sowers.csv` for my custom project.

### Kafka Messages

The producer reads sales records from `sales_sowers.csv` and sends each
record as a Kafka message.

The Kafka topic used for this project is:

`product-sales-case`

The message key is the region ID. This allows messages to be grouped by
geographic region.

I did not change the message structure, but I changed the input and
output file names used by the producer and consumer.

### Consumer Processing

The consumer receives sales transaction messages from Kafka and
processes them one at a time.

For each message, the consumer:

- validates required fields
- enriches the record with calculated values
- calculates subtotal, tax, and total sale amount
- updates running sales statistics
- updates a live visualization
- writes processed records to a CSV file
- checks for high-value sales

The consumer processed six messages and wrote the results to:

`consumed_sales_sowers.csv`

The consumer also generated two visualization files:

- `sales_chart_sowers.png`
- `high_value_sales_sowers.png`

My custom High Value Sale Alert feature logs a warning whenever a sale
total is greater than or equal to $50.00.

### Experiments

#### Phase 4 Modification

For Phase 4, I created custom producer and consumer files named
`kafka_producer_sowers.py` and `kafka_consumer_sowers.py`.

I changed the project to use custom Sowers-specific input and output
files and added a High Value Sale Alert feature that identifies sales
transactions with totals greater than or equal to $50.00.

#### Phase 5 Application

For Phase 5, I created a new visualization named
`high_value_sales_sowers.png`.

The chart displays sales totals along with a threshold line at $50.00,
making it easy to see which transactions triggered High Value Sale
Alerts.

### Results

The producer successfully sent six sales messages to Kafka.

The consumer successfully processed all six messages and generated the
custom output CSV file and both visualization files.

Final results included:

- Messages Consumed: 6
- Messages Skipped: 0
- Total Sales: $471.87
- Average Sale: $78.64
- Minimum Sale: $43.19
- Maximum Sale: $194.82

Five of the six transactions triggered High Value Sale Alerts because
their totals exceeded the configured threshold.

### Interpretation

This project demonstrated how Kafka can be used to stream business
transactions in real time and generate analytics as data arrives.

Compared to the original example, I added custom file names, custom
output files, a High Value Sale Alert feature, and an additional
visualization showing alert thresholds.

Watching messages move through Kafka helped me understand how producers
send data, how consumers process data, and how streaming analytics can
provide immediate feedback.

For a business, this type of workflow could be used to monitor sales
activity, identify important transactions, and track performance trends
in real time.

The business intelligence gained from the consumed messages included
sales totals, average sales, minimum and maximum sales values, and
identification of high-value transactions that may require additional
attention.
